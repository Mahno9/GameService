# Контракт мини-игры

Этот документ описывает всё необходимое для реализации мини-игры, встраиваемой в сервис. Прочитай его полностью перед началом работы.

---

## Что такое мини-игра в контексте сервиса

Мини-игра — это изолированный веб-модуль, который:
- загружается в контейнер на странице меты (глобальной карты) по требованию
- получает конфигурацию снаружи и не имеет собственных настроек в коде
- по завершении передаёт стандартный результат обратно в мету
- после выгрузки не оставляет никаких следов в глобальном состоянии страницы

Мета полностью управляет жизненным циклом игры: запускает, ждёт результата, выгружает.

---

## Структура директории

```
/minigames/
  <game-id>/
    index.js        # точка входа, экспортирует init()
    style.css       # стили (опционально, можно инлайн)
    schema.json     # схема конфигурации для AdminPanel
    assets/         # все ассеты игры (изображения, звуки)
    README.md       # описание игры и её параметров (для людей)
```

`<game-id>` — уникальный строковый идентификатор в kebab-case, например `sliding-puzzle`, `find-object`, `runner`, `arkanoid`.

---

## Точка входа

`index.js` должен экспортировать **одну именованную функцию** `init`:

```js
export function init(container, config, callbacks) { ... }
```

### Аргументы

#### `container: HTMLElement`
DOM-элемент, в который монтируется игра. Игра должна рендериться **только внутри него**. Размеры контейнера задаёт мета — игра подстраивается под них. Не изменяй `document.body`, не добавляй элементы за пределами `container`.

#### `config: object`
Конфигурация игры, сформированная AdminPanel по схеме `schema.json`. Структура произвольная — специфична для каждой игры. Конфиг приходит уже десериализованным объектом. Считай его иммутабельным — не модифицируй.

Пути к ассетам передаются внутри `config` как строки URL (абсолютные, готовые к `fetch` / `src`). Игра не должна конструировать пути самостоятельно.

#### `callbacks: object`
```js
{
  onComplete(result),  // вызвать при нормальном завершении игры
  onExit(),            // вызвать при досрочном выходе игрока
}
```

Ровно один из этих двух колбэков должен быть вызван до того, как игра прекратит работу. Вызывать оба запрещено. После вызова любого из них игра должна прекратить всю активность (таймеры, анимации, обработчики событий) — мета вызовет `destroy()` в ближайший тик.

---

## Объект результата

`onComplete` принимает единственный аргумент — объект результата:

```js
{
  score: number,      // заработанные баллы, целое число >= 0
  won: boolean,       // true = победа, false = поражение
}
```

- `score` всегда обязателен. Если в игре нет понятия «поражение» (игра всегда заканчивается), передавай `won: true`.
- Не добавляй в объект дополнительные поля — мета их игнорирует, но это создаёт путаницу.
- `completedAt` не нужен — мета проставляет timestamp самостоятельно.

---

## Функция destroy

`init` **должна возвращать** объект с методом `destroy`:

```js
export function init(container, config, callbacks) {
  // ... инициализация ...

  return {
    destroy() {
      // остановить все таймеры, RAF-петли, анимации
      // удалить все глобальные обработчики событий (window, document)
      // освободить AudioContext, WebGL-контекст и прочие ресурсы
      // очистить container (container.innerHTML = '' или аналог)
    }
  }
}
```

После вызова `destroy()` контейнер передаётся другому контенту. Утечки памяти и зависшие таймеры недопустимы.

---

## Изоляция — что запрещено

| Запрещено | Почему |
|-----------|--------|
| Изменять `document.title`, `document.body`, `<head>` | Сломает мету |
| Добавлять глобальные CSS (кроме scoped внутри контейнера) | Конфликт стилей |
| Использовать `localStorage`, `sessionStorage`, `IndexedDB` | Персистентность — зона ответственности меты |
| Делать сетевые запросы к сторонним API | Безопасность и оффлайн-режим |
| Читать или писать `window.*` переменные | Конфликт с метой |
| Вызывать `alert()`, `confirm()`, `prompt()` | Блокируют поток, нет доступа в мобильном браузере |
| Регистрировать Service Worker | Конфликт с SW меты |

---

## Звук

- Игра управляет своим звуком самостоятельно через Web Audio API или `<audio>`.
- **Кнопка mute обязательна** — она должна быть видна в UI игры и отключать все звуки игры.
- Глобальный mute меты передаётся через `config.muted: boolean` при старте. Игра должна учесть его при инициализации и не воспроизводить звуки, если `muted = true`.
- Не перехватывай глобальные события громкости.

---

## Переходы

- При монтировании игра должна появляться плавно (fade-in или slide-in, длительность ~300 мс).
- При вызове `onComplete` или `onExit` — плавно скрываться перед тем, как мета вызовет `destroy()`. Допустимо: завершить анимацию победы/поражения, затем вызвать колбэк.

---

## Производительность

- Целевой FPS: **60**. Игра работает на мобильном устройстве пользователя в браузере.
- Используй `requestAnimationFrame` для игрового цикла, не `setInterval`.
- Не блокируй главный поток тяжёлыми вычислениями — выноси в Web Worker если нужно.
- Ассеты загружай лениво — только в момент запуска игры, не раньше. Пути уже есть в `config`.
- Не храни тяжёлые объекты в замыканиях после `destroy()`.

## Файл schema.json

Описывает конфигурацию игры. AdminPanel читает его и автоматически генерирует форму настройки. Формат — JSON Schema (draft-07) с расширениями:

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Название игры",
  "description": "Краткое описание",
  "type": "object",
  "properties": {
    "lives": {
      "type": "integer",
      "title": "Количество жизней",
      "default": 3,
      "minimum": 1
    },
    "backgroundImage": {
      "type": "string",
      "title": "Фоновое изображение",
      "x-type": "asset:image"
    },
    "rounds": {
      "type": "array",
      "title": "Раунды",
      "items": { ... }
    }
  },
  "required": ["lives", "backgroundImage"]
}
```

### Расширение `x-type`

Указывает AdminPanel, какой виджет рендерить для поля:

| Значение `x-type` | Виджет |
|-------------------|--------|
| `asset:image` | Загрузчик изображения, возвращает URL |
| `asset:audio` | Загрузчик аудиофайла, возвращает URL |
| `asset:gif` | Загрузчик GIF или изображения, возвращает URL |
| `curve` | Редактор кривой (список точек `{x, y}`) |
| `weighted-list` | Список с весами (массив `{value, weight}`) |

Поля без `x-type` рендерятся стандартными контролами по типу JSON Schema (`string` → текстовое поле, `integer` → числовое, `boolean` → чекбокс, `array` → повторяемая группа).

## Запуск в изолированном режиме (тестирование)

AdminPanel может запустить игру изолированно, без меты. В этом случае:
- `container` — полноэкранный `<div>`
- `config` — значения из формы AdminPanel
- `callbacks.onComplete` и `callbacks.onExit` — заглушки (логируют результат в консоль)

Игра **не должна знать**, запущена ли она из меты или в тестовом режиме. Поведение должно быть идентичным.

- [ ] Кнопка mute присутствует в UI игры

- [ ] `schema.json` описывает все параметры игры
- [ ] FPS стабильно 60 на среднем мобильном устройстве
- [ ] Игра корректно работает при запуске из AdminPanel в изолированном режиме
- [ ] Все ассеты загружаются из путей в `config`, не захардкожены
