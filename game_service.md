# Техническая спецификация: Сервис навигации с игровыми точками интереса

---

## 1. Общее описание

Сервис представляет собой персональный квест-подарок: один игрок физически перемещается по заданному участку местности, находит точки интереса на карте, проходит привязанные к ним мини-игры, получает предметы-награды и по итогу занимает место в таблице лидеров среди фиктивных участников.

---

## 2. Инфраструктура и развёртывание

### Ресурсы
- CPU: 2 ядра
- RAM: 2 ГБ
- Хранилище: до 10 ГБ

### Развёртывание
- Весь сервис упакован в Docker Compose: приложение + база данных + туннель
- Для публикации с домашнего сервера — CloudFlare Tunnel (`cloudflared`) как отдельный сервис в Compose
- HTTPS обязателен (браузер не выдаёт GPS без HTTPS)

### HTTPS и домен
- Зарегистрировать домен на Cloudflare (~$10/год)
- `cloudflared` в Docker Compose туннелирует трафик с `game.домен.com` на `localhost:порт`
- Сертификат выдаётся Cloudflare автоматически
- Порты на роутере открывать не требуется

### Объём данных
- Сервис должен вписываться в 10 ГБ хранилища, включая: предобработанные тайлы карты, ассеты мини-игр, загруженные изображения, базу данных

---

## 3. Карта

### Источник данных
- OpenStreetMap (топология, здания с высотами, пешеходные и велосипедные маршруты)
- В качестве подложки в AdminPanel для выбора участка — OSM-тайлы стандартного вида

### Два режима отображения

| Режим   | Вид на экране               | Формат хранения       | Рендеринг                          |
|---------|-----------------------------|-----------------------|------------------------------------|
| Близкий | 50–300 м по ширине экрана   | Векторные тайлы (MVT) | На устройстве пользователя, WebGL  |
| Дальний | 1/10 участка — весь участок | Растровые тайлы (WebP)| Сервер отдаёт готовые изображения  |

Переключение между режимами — по уровню зума, порог настраивается в AdminPanel.

---

### Близкий режим — векторный 3D (клиентский рендеринг)

Хранится как **векторные тайлы MVT** (Mapbox Vector Tiles, формат protobuf). Рендеринг выполняется на устройстве пользователя через WebGL, что обеспечивает:
- свободное вращение карты по направлению движения без артефактов
- экструзию зданий в реальном 3D по данным `height` / `building:levels` из OSM
- динамические тени, ambient occlusion
- плавное масштабирование без пикселизации

**Стилизация** задаётся через JSON-файл стиля (совместимый с MapLibre GL Style Spec), хранится на сервере. Допускает кастомизацию цветов, шрифтов, иконок POI, степени экструзии.

**Библиотека рендеринга:** MapLibre GL JS (открытый форк Mapbox GL JS, без API-ключей).

---

### Дальний режим — растровые тайлы (предгенерация)

Хранится как растровые тайлы WebP, сгенерированные заранее в браузере AdminPanel. Вращение реализуется на стороне клиента трансформацией поверх растра (без перегенерации тайлов).

---

### Предобработка (выполняется в браузере AdminPanel)

Вся тяжёлая обработка выполняется **в браузере AdminPanel**, не на сервере. Сервер только скачивает сырые данные и хранит результат.

**Поток:**
1. AdminPanel запрашивает сырые OSM-данные — сервер скачивает их с Overpass API и проксирует как есть
2. Браузер парсит OSM-данные и генерирует:
   - **Векторные MVT-тайлы** для близкого режима (нарезка геометрии по тайловой сетке, упаковка в protobuf)
   - **Растровые WebP-тайлы** для дальнего режима (рендер через Canvas, стилизация)
3. Готовые тайлы загружаются на сервер пакетами; сервер сохраняет в файловую систему
4. Прогресс отображается в AdminPanel в реальном времени

**WebAssembly** — операции, выносимые в WASM-модуль для ускорения:
- Парсинг OSM PBF (бинарный protobuf)
- Нарезка геометрии на тайловую сетку (clip, simplify по zoom-уровню)
- Упаковка MVT (protobuf serialization)
- Кодирование растровых тайлов в WebP

WASM-модуль компилируется заранее, поставляется вместе с фронтендом AdminPanel. Если WASM недоступен — JS-fallback (медленнее, но работоспособно).

---

### Навигация пользователя
- Положение игрока — по GPS устройства
- Карта вращается по направлению движения (heading из GPS/компаса) в **обоих режимах**
- Угол поворота сохраняется при переходе между близким и дальним режимами — переключение происходит без сброса ориентации
- В близком режиме вращение реализовано через WebGL-трансформацию MapLibre
- В дальнем режиме вращение реализовано через CSS/Canvas-трансформацию растрового слоя (rotate вокруг центра viewport)
- Зум: два режима, переключение жестом pinch или кнопкой

### Хранение и отдача тайлов
- MVT-тайлы: `/{z}/{x}/{y}.mvt` — отдаются с Content-Type `application/x-protobuf`
- Растровые тайлы: `/{z}/{x}/{y}.webp` — отдаются как статика
- Оба набора: Cache-Control: max-age=31536000
- Обновление — повторная предобработка из AdminPanel (старые тайлы перезаписываются)

## 4. Точки интереса (POI)

- Каждая POI имеет: **название**, координаты, назначенную мини-игру, список блокирующих POI (должны быть пройдены первыми), флаг возможности переигрывания, описание награды (изображение + текст + название предмета при победе + название предмета при поражении)
- Название POI отображается на карте под иконкой и используется в AdminPanel для идентификации точки в списках
- Радиус триггера — глобальная настройка AdminPanel, применяется ко всем POI одинаково

**Поведение при клике на POI:**
- *Игрок далеко (вне радиуса):* клик на иконку открывает информационную карточку — название игры, краткое описание, статус (заблокирована / пройдена / доступна). Начать игру нельзя
- *Игрок в радиусе:* POI пульсирует / подсвечивается; клик открывает подтверждающий попап с названием игры и кнопкой «Начать» — игрок сам решает, хочет ли запустить игру прямо сейчас
- Пока игрок находится в радиусе, POI остаётся кликабельной (в том числе для переигровки без необходимости уходить и возвращаться)
- При выходе из радиуса подсветка гасится; клик снова открывает только информационную карточку
- На карте POI отображаются иконками с состоянием: заблокирована / доступна / в радиусе (кликабельна) / пройдена и доступна для переигровки / пройдена

## 5. Пользователь и сессия

### Регистрация
- При первом запуске: ввод имени + выбор аватарки из набора эмоджи (лица и животные)
- Никакого пароля, никакой почты

### Персистентность
- Состояние игры (прогресс, баллы, предметы, позиция) хранится локально (IndexedDB или localStorage)
- Синхронизация с сервером: каждые 30 секунд (настраивается в админ-панели) + при ключевых событиях (завершение игры, получение предмета, начало сессии)
- При переоткрытии браузера: сессия восстанавливается из локального состояния, при расхождении — берётся более новое по timestamp

### Мультидевайс
- На втором устройстве после ввода того же имени загружается последнее синхронизированное состояние
- Конфликты разрешаются по timestamp последней синхронизации

---

## 6. Игровой процесс и баллы

### Порядок прохождения
- Игра может быть заблокирована одной или несколькими другими POI: пока они не пройдены, игра недоступна
- Временного ограничения на весь маршрут нет

### Переигрывание
- Каждая мини-игра имеет параметр `replayable` (bool, по умолчанию `false`)
- Если `replayable = true`, игрок может пройти игру повторно при возврате к POI
- В зачёт идёт наибольший результат из всех попыток
- Итоговые баллы в таблице лидеров пересчитываются при каждом улучшении

---

## 7. Награды и инвентарь

- По завершении каждой игры игроку выдаётся предмет (всегда, независимо от результата)
- Предмет содержит: изображение, название, описание
- Название меняется в зависимости от результата: «Награда» (победа) / «Утешительный приз» (поражение) — для игр, где возможно поражение
- Предмет выдаётся один раз; при переигрывании повторно не выдаётся
- **Инвентарь**: отдельный экран со всеми полученными предметами, открывается по кнопке в UI; кнопка размещается в углу экрана, подальше от элементов навигации и управления картой

---

## 8. Таблица лидеров

- Становится доступна после прохождения всех игр
- Кнопка на основном экране (не блокирует карту)
- Игрок вставляется в список фиктивных участников согласно набранным баллам
- Фиктивные участники настраиваются в админ-панели: имя, аватарка (эмоджи), количество баллов
- Количество фиктивных записей — произвольное, настраивается в панели
- Результаты дебаг-сессий могут быть удалены из таблицы лидеров (см. п. 11)

---

## 9. UI глобальной карты (меты)

- Все интерактивные элементы на экране карты (кнопки инвентаря, таблицы лидеров, зума, компаса, переключения режима и любые другие) воспроизводят звуковой эффект нажатия
- Звуки кнопок настраиваются как ассеты меты (отдельно от ассетов мини-игр)
- Предусмотреть глобальную кнопку mute, отключающую все звуки сервиса (мета + мини-игры)

---

## 10. Локализация

- Поддерживаемые языки: **русский**, **английский**
- Переключение языка — в настройках пользователя
- Все текстовые строки (включая описания игр, награды, UI) должны быть вынесены в файлы локализации

---

## 10. Оффлайн и потеря GPS

### Потеря GPS
- Если GPS не определяется более N минут (настраивается в админ-панели, по умолчанию 1 минута): предлагается включить навигацию через виртуальный джойстик (как в дебаг-режиме)
- Уведомление не блокирует игру

### Потеря интернета
- Если необходимые ресурсы загружены (тайлы карты, ассеты игры), игра продолжается в оффлайне
- Синхронизация ставится в очередь и выполняется при восстановлении соединения
- На экране отображается ненавязчивый индикатор отсутствия связи
- Недоступный контент (незагруженные тайлы, ассеты) помечается как недоступный

---

## 11. Дебаг-режим

- Включается/выключается глобально из админ-панели (влияет на всех пользователей сервиса)
- Пользователь видит виртуальный джойстик на экране для перемещения по карте без физического передвижения
- Скорость перемещения джойстика настраивается в AdminPanel
- Весь игровой процесс функционирует в полном объёме (игры, баллы, награды, таблица)
- При выходе из дебаг-режима: уведомление с предложением удалить результаты тестовой сессии
- Удалить тестовые результаты также возможно позднее через таблицу лидеров в AdminPanel

---

## 12. Административная панель

### Аутентификация
- Один захардкоженный аккаунт (логин + пароль в конфигурации сервиса)

### Разделы

#### 12.1 Карта
- Выбор прямоугольного участка для предобработки на OSM-подложке (стандартные тайлы OSM в качестве подложки для навигации по карте)
- Запуск генерации тайлов (близкий и дальний наборы) — выполняется в браузере AdminPanel (см. п. 3 «Клиентская генерация тайлов»)
- Индикатор прогресса: количество обработанных тайлов / всего, текущий этап (загрузка OSM → парсинг → рендер → загрузка на сервер)
- Оценка времени до завершения
- Возможность прервать и возобновить генерацию (чекпойнты по зум-уровням)
- Возможность перегенерации уже готового набора тайлов

#### 12.2 Точки интереса
- Расстановка POI на карте кликом
- Для каждой POI: **название** (отображается на карте и в списках AdminPanel), координаты, назначение мини-игры, список блокирующих POI, флаг `replayable`
- Описание награды POI: изображение, название предмета при победе, название предмета при поражении (если применимо), текстовое описание предмета
- Перетаскивание POI для смены координат
- Глобальная настройка радиуса триггера (применяется ко всем POI)

#### 12.3 Настройка мини-игр
- Список всех POI с назначенными играми — каждая строка показывает **название POI** и тип игры
- По клику на строку — всплывающее окно с параметрами игры (специфичные для каждой, см. п. 13); заголовок окна — название POI
- В каждом окне: поля баланса, флаг `replayable`; настройка награды — в разделе 12.2 (POI)
- Кнопка «Запустить игру в тестовом режиме» (запускает игру изолированно, без привязки к карте)

#### 12.4 Дебаг-режим
- Тумблер включения/выключения глобального дебаг-режима
- Настройка скорости виртуального джойстика
- Настройка таймаута потери GPS (минуты)
- Настройка интервала синхронизации (секунды)

#### 12.5 Таблица лидеров
- Список фиктивных участников: добавление/удаление/редактирование (имя, аватарка-эмоджи, баллы)
- Удаление результатов дебаг-сессий из таблицы

#### 12.6 Предпросмотр
- Режим просмотра карты «глазами игрока»: полный UI с GPS-следованием (или джойстиком)
- Упрощённый режим обзора карты: просмотр расположения всех POI, их состояния, радиусов триггеров

## 13. Мини-игры

### Общие требования к мини-играм

- Каждая мини-игра — изолированный модуль (загружается по требованию, выгружается после закрытия)
- Загрузка происходит только при запуске игры; до этого момента ассеты не загружаются
- Модуль может запускаться изолированно (из AdminPanel или для тестирования)
- **Стандартизированный результат**: каждая игра по завершении передаёт объект:
  ```
  {
    score: number,         // заработанные баллы
    won: boolean,          // победа или поражение (для названия предмета)
    completedAt: timestamp
  }
  ```
- Внутренняя схема данных конфигурации игры — произвольная, специфична для каждой игры
- Каждая игра поддерживает звуковые эффекты (опционально, с кнопкой mute)
- Переход в/из игры — плавный (fade или slide)

---

### 13.1 Пятнашки

**Описание:** Классические пятнашки из заданного изображения, несколько раундов.

**Параметры в AdminPanel:**
- Размер сетки (число, по умолчанию 4×4)
- Список раундов. Для каждого раунда:
  - Изображение
  - Параметр сложности перемешивания: `shuffle_moves` (количество ходов перемешивания от решённого состояния). Примечание: точная нижняя граница числа ходов не вычисляется — параметр определяет глубину перемешивания
  - Список порогов баллов по времени: `[{ max_seconds: N, points: P }, ...]`

**Ассеты:**
- Изображения для каждого раунда (загружаются в AdminPanel)
- Звуки: ход плитки, победа в раунде, общая победа

**Геймплей:**
- Пользователь перетаскивает плитки
- По завершении раунда: анимация сборки → окно поздравления + звук → тап → следующий раунд
- Баллы за игру = сумма баллов всех раундов

---

### 13.2 Найди предмет

**Описание:** Игрок ищет заданные предметы на большом изображении (аналог «Где Уолдо»).

**Параметры в AdminPanel:**
- Фоновое изображение (до 4K; при загрузке нарезается на тайлы для эффективного отображения)
- Список оверлей-изображений (декор, «шум»): координаты, поворот, цветовой фильтр, порядок слоёв
- Упорядоченный список целевых предметов (target list): координаты, поворот, цветовой фильтр, порядок слоёв, изображение
- Порог баллов по времени: `[{ max_seconds: N, points: P }, ...]`

**Редактор расположения объектов (в AdminPanel):**
- Drag-and-drop размещение изображений на фоне
- Настройка поворота и цветового фильтра для каждого объекта
- Управление порядком слоёв (z-index) для всех объектов (целевых и декоративных)
- Предпросмотр итогового вида

**Ассеты:**
- Фоновое изображение
- Изображения оверлеев
- Изображения целевых предметов
- Звуки: нахождение предмета, победа, фоновая музыка (опционально)

**Геймплей:**
- Игроку показываются 3 верхних предмета из target list
- Игрок может скроллить и зумировать фон
- **Любой предмет** (и целевой, и декоративный) можно перетаскивать зажатием:
  - При зажатии и начале переноса: анимация «поднятия» — масштаб чуть увеличивается, появляется тень
  - При отпускании: анимация «приземления» — предмет чуть сжимается и возвращается к нормальному размеру, тень исчезает
- Тап по предмету без перетаскивания: если попадание в область изображения целевого предмета — засчитывается
  - Если целевой предмет перекрыт другим: необходимо сначала перетащить верхний в сторону
- **Эффект нахождения предмета** (визуальный + звуковой):
  - Предмет увеличивается в размере
  - Появляется яркая обводка вокруг контура
  - Предмет становится ярче (всплеск яркости/насыщенности)
  - Растворяется в искорках (particle burst)
  - Одновременно воспроизводится звуковой эффект нахождения
- После нахождения предмет исчезает из списка, следующий из очереди становится активным
- Игра заканчивается, когда весь target list исчерпан
- Баллы зависят от итогового времени прохождения

---

### 13.3 Прыгалка

**Описание:** Бесконечный раннер, персонаж прыгает через препятствия (аналог динозаврика Chrome).

**Параметры в AdminPanel:**
- Количество жизней
- Кривая роста скорости (список точек `[{ distance: N, speed: V }, ...]`)
- Ассет персонажа (изображение или гифка)
- Список ассетов фоновых слоёв (parallax): изображение + скорость прокрутки
- Список типов препятствий. Для каждого типа:
  - Ширина, высота (игровые единицы)
  - Относительная частота появления (вес, число; чем больше — тем чаще выпадает относительно других типов)
  - Ассет по умолчанию (изображение или гифка)
  - Звук преодоления (опционально)
  - Альтернативный ассет при преодолении (опционально: изображение или гифка, воспроизводится в момент прыжка через препятствие)
- Фоновая музыка (опционально)
- Звуки: прыжок, приземление, потеря жизни, игра окончена

**Ассеты:**
- Ассет персонажа (idle, run, jump, crouch, hit)
- Ассеты фона (один или несколько слоёв parallax)
- Ассеты препятствий (см. выше)
- Звуки и музыка

**Геймплей:**
- Тап по экрану — прыжок
- Двойной тап или зажатие — приседание
- Потеря жизни при столкновении с препятствием
- Баллы = пройденное расстояние
- Игра заканчивается при потере всех жизней

---

### 13.4 Арканоид

**Описание:** Классический арканоид с бонусами, несколькими уровнями и современными визуальными эффектами.

**Параметры в AdminPanel:**
- Количество жизней (глобально для всей игры)
- Список уровней. Для каждого уровня:
  - Изображение-подложка (фон за блоками)
  - Общее количество блоков
  - Список типов ценных блоков:
    - Цвет блока
    - Процент от общего числа блоков
    - Стоимость (баллы)
    - Время жизни (секунды до потери ценности)
  - Параметры скорости мяча и ускорения

**Бонусы и бустеры (встроенные):**
- Расширение платформы
- Сужение платформы
- Ускорение/замедление мяча
- Дополнительный мяч
- Лазерная платформа (стреляет по блокам)
- Щит (однократный отбив мяча у нижнего края)
- Взрыв (уничтожает блоки вокруг)
- Замедление времени

**Ассеты:**
- Изображения фона для каждого уровня
- Ассеты мяча и платформы
- Анимации бонусов
- Звуки: отбив мяча, разрушение блока, подбор бонуса, потеря мяча, победа, поражение
- Фоновая музыка (опционально)

**Геймплей:**
- Управление: перетаскивание пальцем в любом месте экрана ИЛИ наклон устройства
- На экране во время игры: легенда «цвет блока — стоимость» для текущего уровня
- Потеря мяча за нижним краем = потеря жизни
- Уровень пройден при уничтожении всех блоков
- Игра заканчивается при потере всех жизней
- Баллы = сумма стоимостей всех разбитых ценных блоков по всем уровням

## 14. Архитектура мини-игр (для разработчиков)

В репозитории предусмотрен каталог `/minigames/`. Каждая мини-игра — отдельная директория.

Полный контракт мини-игры (жизненный цикл, изоляция, формат результата, схема конфигурации, чеклист) вынесен в отдельный документ: **[minigame_contract.md](minigame_contract.md)**

Краткая выжимка:
- Экспортирует `init(container, config, callbacks)`, возвращает `{ destroy() }`
- Вызывает `callbacks.onComplete({ score, won })` или `callbacks.onExit()` — ровно один раз
- Не трогает ничего за пределами `container`, не использует `localStorage`, не регистрирует глобальные обработчики без последующей очистки
- Конфигурационная схема описывается в `schema.json` — AdminPanel генерирует форму по ней автоматически

## 15. Требования к производительности

- Целевой FPS мини-игр: 60 (при 2 ядрах и 2 ГБ RAM вся нагрузка на клиентский браузер)
- Серверная часть должна потреблять не более 300 МБ RAM в рабочем состоянии
- Тайлы карты отдаются как статика с заголовками кэширования (Cache-Control: max-age=31536000)
- Ассеты мини-игр загружаются только при запуске соответствующей игры (lazy load)
- Изображения в AdminPanel при загрузке проходят автоматическую оптимизацию (WebP, тайлинг для 4K)
